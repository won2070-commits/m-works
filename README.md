# M WORKS — 들리는 설교, 끌리는 설교

> 구 이름: 말씀공방 (폴더·프리뷰 이름은 말씀공방 유지). 바탕화면 배포물: **M WORKS.app**(더블클릭 버튼), M WORKS_앱.html(단일 파일).

『들리는 설교, 끌리는 설교』(도원욱)의 방법론을 실제 설교 준비 과정으로 구현한 웹 앱.
AI가 설교를 대신 쓰는 도구가 아니라, 설교자가 본문을 연구하고 중심사상을 발견하고
원고를 짓고 연습하도록 돕는 **설교 작업 공간**이다.

## 실행 (세 가지 방법)

1. **`말씀공방_앱.html` 더블클릭** — 서버 없이 열리는 단일 파일. AI는 설정에서 API 키 입력 시 직접 호출. (바탕화면에도 사본 있음)
2. **`말씀공방 실행.command` 더블클릭** — 서버를 켜고 브라우저를 여는 전체 버전. 터미널 claude 로그인만 돼 있으면 API 키 없이 AI 연결.
3. 터미널: `node 말씀공방/server.cjs` → http://localhost:5208 (Cowork 프리뷰명 "말씀공방")

수정 후 단일 파일 재생성: `node 말씀공방/build.cjs`

## AI 연결 (둘 중 하나, 최초 1회)

1. **Claude 로그인(권장)** — `AI 연결하기.command` 더블클릭 → 안내대로 `/login` → 브라우저 승인. 이후 `말씀공방 실행.command`로 열면 AI 자동 연결.
2. **API 키** — 앱 설정에서 Anthropic API 키 입력(브라우저에만 저장), 또는 서버 환경변수 `ANTHROPIC_API_KEY`. 단일 파일(`말씀공방_앱.html`)은 이 방법만 지원.

연결 상태는 좌측 하단 배지와 설정 화면에서 확인. 미연결이어도 수동 작성 경로(2단계 "AI 없이 직접 작성", 3단계 "빈 원고에서 직접 쓰기")로 전 과정을 쓸 수 있다.

## 구조

| 파일 | 역할 |
|---|---|
| `server.cjs` | 정적 서빙 + `/api/status` + `/api/ai`(SSE 스트리밍). Anthropic API 또는 claude CLI 중계 |
| `prompts.js` | 단계별 AI 프롬프트 10종 + 설교 형식 카탈로그 9종. 책의 방법론 요약이 시스템 프롬프트에 내장 |
| `app.js` | 5단계 화면, 문서 편집기, 프로젝트 보관함, 버전 기록, 리허설 모드, 내보내기, 설정/프롬프트 관리 |
| `index.html` / `style.css` | 서재풍 UI (짙은 녹색·크림·금색, Noto Serif KR) |

## 다섯 단계

1. **본문 찾기** — 입력 본문 확정 + 단락 확인(AI), 또는 주제·상황·필요 분석으로 후보 5개 추천(+2~3개 추가). 본문 전문은 저작권·환각 방지를 위해 사용자가 성경에서 붙여넣는다.
2. **중심사상** — 로빈슨 12단계 분석(문학 단위→문맥→…→주요소→보조요소→석의·신학·설교적 세 문장→FCF). 모든 문장 직접 수정 가능 + 정련 버튼 6종.
3. **설교 작성** — 목표 시간 기준 초안 생성(스트리밍), Word형 편집기(스타일·강세·멈춤·시선·메모 표시, 낭독 시간, 버전 기록·비교), 부분 재작성 21종.
4. **형식 결정** — 기본 9형식(원포인트/삼대지/연역/귀납/내러티브/로우리/네 페이지/먼로/문제-원인-복음-적용) + 사용자 정의 형식. 변환 전후 개요 비교 → 승인 시 반영(이전 원고는 버전 보관).
5. **연습하기** — 내용+전달 피드백(AI), 제스처 5~10 제안(AI), 리허설 모드(큰 글씨·문단 넘기기·타이머·기록), 최종 점검표 8항.

## 저장·내보내기

- 저장: 브라우저 localStorage `malsseum_v1` (자동 저장 + 버전 기록 + 휴지통 + JSON 전체 백업/가져오기)
- 내보내기: Word `.doc`(아래한글에서도 열림), `.txt`, `.md`, 인쇄용, 강단용 큰 글씨. 포함 항목 선택 가능.
- `.hwp` 직접 생성은 하지 않음(정식 `.hwpx`는 다음 버전 계획).

## 알려진 한계 (1차 버전)
